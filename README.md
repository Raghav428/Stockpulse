# StockPulse Platform

StockPulse is a real-time stock data ingestion, storage, querying, and alerting platform. It's built with modern technologies to efficiently handle high-frequency time-series data while providing a robust backend for user management and live updates.

## Architecture

```text
Browser
  → nginx (SSL termination, reverse proxy, load balancing)
    → FastAPI service (async, handles HTTP + WebSocket)
      → Redis (live intraday data, derived cache)
      → Cassandra (historical tick data)
      → PostgreSQL (users, watchlists, alerts)

Ingestion Pipeline (Runs independently):
  External API (Yahoo/NSE/BSE)
    → Ingestion Service (fetches + publishes)
      → Kafka (event buffer, source of truth for tick data)
        → Consumer Service (reads Kafka, writes to Redis + Cassandra)
```

## Current Features
- **Dockerized Environment**: Fully containerized with PostgreSQL, Redis, and FastAPI.
- **Async Framework**: FastAPI utilizing asynchronous endpoints and SQLAlchemy configurations.
- **Autogenerated Migrations**: Integration with Alembic for auto database migrations.
- **User Authentication**: Secure password hashing with Argon2 and JWT token access (sign-up/login functionality implemented, along with OAuth2 protected routes).
- **Data Models**: Foundation for relational data structures including Users and Watchlists using SQLAlchemy.
- **Environment Management**: Injection of secrets across containers using Docker Compose.

## Quick Start
To spin up the development environment, review the `setup_manual.txt` for Docker Compose commands. 

- Build and start all services: `docker compose up -d --build`
- Check logs: `docker compose logs -f fastapi`
- Stop and reset: `docker compose down -v`

For deeper understanding of the codebase and project structure, please see `stockpulse_handoff.md`.
